# 핀테크트라이브 위클리 요약

대상 날짜: `$ARGUMENTS` (인자가 없으면 오늘 날짜 !`date +%Y-%m-%d` 사용)

## 실행 절차

### Step 1: Confluence에서 대상 페이지 찾기

#### 1-1. 개발팀 위클리 페이지 찾기

개발팀 위클리 폴더 parentId: `5565612039`, site: `kurly0521.atlassian.net`

1. `getConfluencePageDescendants`로 하위 페이지 목록 조회 (depth: 1, limit: 10)
2. 제목 형식: `[개발] 26년 MM/DD - 26년 MM/DD`
3. 대상 날짜가 제목의 날짜 범위에 포함되는 페이지를 찾는다
4. `getConfluencePage` (contentFormat: markdown)으로 본문을 읽는다

#### 1-2. 기획팀 위클리 페이지 찾기

기획팀 위클리 폴더 parentId: `3963650997`, site: `kurly0521.atlassian.net`

1. `getConfluencePageDescendants`로 월별 하위 폴더 목록 조회 (depth: 1, limit: 5)
2. 대상 날짜의 월에 해당하는 폴더를 찾는다 (예: `26.04월 핀테크 트라이브 기획파트`)
3. 해당 월 폴더의 `getConfluencePageDescendants`로 주별 페이지 목록 조회 (depth: 1, limit: 10)
4. 제목 형식: `[기획]'26년 M/DD-M/DD`
5. 대상 날짜가 제목의 날짜 범위에 포함되는 페이지를 찾는다
6. `getConfluencePage` (contentFormat: markdown)으로 본문을 읽는다

### Step 2: 향후 2주내 배포 예정 정보 생성

개발팀 위클리 페이지의 테이블에서 **"향후 2주내, 예정된 주요 배포 내역"** 컬럼을 파싱한다.

- 대상 날짜 기준으로 과거 배포일은 "배포완료", 미래 배포일은 "배포예정"으로 상태 표기
- 배포일 기준 오름차순 정렬

출력 포맷:
```markdown
## 향후 2주내 배포 예정

| 배포일 | 배포 기능 | 담당자 | 서비스 | 상태 |
|--------|--------|--------|-----------|------|
| **MM-DD** | 기능 설명 | 담당자명 | 서비스명 | 배포완료 또는 배포예정 |
```

### Step 3: 과제 이름별 진행 사항 통합 정리

개발팀 위클리 페이지의 **"수행 과제"** 컬럼과 기획팀 위클리 페이지의 **"금주완료"** 컬럼을 각각 파싱한다.

**핵심: 과제 이름별로 개발/기획 양쪽 내용을 통합 정리한다.**

1. 개발팀 "수행 과제"에서 과제 이름과 상태를 추출
2. 기획팀 "금주완료"에서 과제 이름과 상태를 추출
3. 동일하거나 유사한 과제 이름을 기준으로 그룹핑하여 정리

#### 3-1. 도메인별 그룹핑

과제를 아래 도메인 카테고리로 분류하여 `### 카테고리명` 헤딩 아래에 배치한다:
- **정산**: 차액정산, 공급사 상계 등 정산 관련
- **결제/카드**: 간편등록, PLCC, 빠른결제, pg-front, 카드이미지, 주문서 등 결제 관련
- **캐시/선불**: 캐시/상품권 분리, 쿠폰 연동, 컬리페이 리뉴얼 등 선불 관련
- **컴플라이언스/인프라**: 마이데이터, SAP, AML, Framework Upgrade, 그레이망 이관 등

#### 3-2. 과제별 출력 포맷

**기획+개발 양쪽 모두 내용이 있는 과제** (순수 markdown 테이블 사용):
```markdown
#### <span style="color:#2563EB">**[과제명]**</span> `상태뱃지`  기획-OOO, 개발-OOO

| 기획팀 | 개발팀 |
|------|------|
| (기획팀 금주완료 내용 요약) | (개발팀 수행과제 내용 요약) |
```

**한쪽 팀만 내용이 있는 과제** (불릿리스트 사용):
```markdown
#### <span style="color:#2563EB">**[과제명]**</span> `상태뱃지`  개발-OOO

- 내용1
- 내용2
```

과제명 스타일 규칙:
- 반드시 `#### <span style="color:#2563EB">**과제명**</span>` 형식으로 h4 헤딩 + 굵은 파란색 적용
- HTML 테이블 사용 금지, 순수 markdown 사용

상태 뱃지 규칙:
- 과제 헤딩 옆에 백틱으로 감싼 짧은 상태 텍스트 추가
- 예시: `QA`, `개발중`, `분석중`, `검토중`, `대기`, `협의중`, `계획중`, `진행중`, `착수`, `배포예정`, `후속대기`
- 과제의 전체적인 진행 상태를 가장 잘 나타내는 한 단어로 요약

내용 줄바꿈 규칙:
- 과제별 통합 테이블(기획팀/개발팀)은 여러 항목을 **각각 별도 행(row)**으로 분리하여 작성
- 과제별 통합 테이블은 HTML 줄바꿈(`<br>`, `<br/>`)을 사용하지 않는다.
- 기획팀/개발팀 항목 수가 다를 경우, 적은 쪽의 빈 셀은 공백으로 처리
- 작성자별 테이블은 다중 항목을 콤마(,) 기준으로 분리하여 `* 항목` 형식으로 표기한다.
- 작성자별 테이블에서 같은 셀 안 다중 항목은 `<br>`로 줄바꿈한다.
- 예시:

| 기획팀 | 개발팀 |
|------|------|
| 기획항목1 | 개발항목1 |
| 기획항목2 | 개발항목2 |
| 기획항목3 | |

#### 3-3. 작성자별 상세 내용

그 아래에 작성자별 상세 내용도 포함한다.

업무강도 표기 규칙:
- 업무강도는 숫자만 **볼드** 처리해서 표기한다.
- 막대 그래프 표시는 사용하지 않는다.

```markdown
## 개발팀 작성자별 수행 과제

### 이름 (파트) **140**

| 과제 | 상태 |
|------|------|
| ... | ... |

## 기획팀 작성자별 금주 완료

### 이름 **100**

| 과제 | 상태 |
|------|------|
| ... | ... |
```

특이사항 (퇴사, 신규입사, 휴가 등)은 이름 뒤에 `— 4/17 퇴사`, `— 신규 입사`, `— 휴가 3/30 ~ 4/24` 형태로 표기한다.

## 주요 보고 사항

개발팀/기획팀 위클리 내용을 분석하여 아래 5개 카테고리로 분류하여 요약한다.
각 카테고리에 해당 사항이 없으면 "없음"으로 표기한다.

### ❶ 의사결정 요청
- 경영진/리더 의사결정이 필요한 사항 (기획 착수 여부, 일정 확정, 예산 등)

### ❷ 협업 요청
- 타 팀/파트와 협업이 필요하거나 진행 중인 사항 (요청 대기, 회신 대기 등)

### ❸ 리스크/이슈
- 장애, 퇴사/인수인계, 데이터 이슈, 일정 지연 등 리스크 항목
- **[장애]**, **[퇴사]**, **[데이터]**, **[일정]** 등 태그로 구분

### ❹ 액션아이템 업데이트
- **[오늘]**, **[이번주]**, **[다음주]** 등 시점 태그와 함께 주요 마일스톤/배포/QA 일정

### ❺ 주요 공유 & 공지사항
- 미팅, 교육, 공지, 계약 등 팀 전체 공유 사항
