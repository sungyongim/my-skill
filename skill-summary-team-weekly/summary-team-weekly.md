# Summary Team Weekly — 핀테크트라이브 위클리 요약

대상 날짜: `$ARGUMENTS` (인자가 없으면 오늘 날짜 사용)

핀테크트라이브 개발/기획팀 위클리를 요약하여 로컬 파일로 저장한다.

## 실행 절차

### Step 1: Confluence에서 대상 페이지 찾기

#### 1-1. 개발팀 위클리 페이지 찾기

개발팀 위클리 폴더 parentId: `5565612039`, site: `kurly0521.atlassian.net`

1. `getConfluencePageDescendants`로 하위 페이지 목록 조회 (depth: 1, limit: 10)
2. 제목 형식: `[개발] 26년 MM/DD - 26년 MM/DD`
3. 대상 날짜가 제목의 날짜 범위에 포함되는 페이지를 찾는다
4. `getConfluencePage` (contentFormat: markdown)으로 본문을 읽는다

#### 1-2. 기획팀 위클리 페이지 찾기

기획팀 위클리 폴더 parentId: `3963650997`, site: `kurly0521.atlassian.net`

1. `getConfluencePageDescendants`로 월별 하위 폴더 목록 조회 (depth: 1, limit: 5)
2. 대상 날짜의 월에 해당하는 폴더를 찾는다 (예: `26.04월 핀테크 트라이브 기획파트`)
3. 해당 월 폴더의 `getConfluencePageDescendants`로 주별 페이지 목록 조회 (depth: 1, limit: 10)
4. 제목 형식: `[기획]'26년 M/DD-M/DD`
5. 대상 날짜가 제목의 날짜 범위에 포함되는 페이지를 찾는다
6. `getConfluencePage` (contentFormat: markdown)으로 본문을 읽는다

### Step 2: 향후 2주내 배포 예정 정보 생성

개발팀 위클리 페이지의 테이블에서 **"향후 2주내, 예정된 주요 배포 내역"** 컬럼을 파싱한다.

- "배포 예정"으로 표기된 항목을 우선 정리
- "배포 완료"로 표기된 항목은 별도 섹션으로 분리
- 배포일 기준 오름차순 정렬

출력 포맷:
```markdown
## 향후 2주내 배포 예정

| 배포일 | 상태 | 서비스 | 기능 | 담당 |
|--------|------|--------|------|------|
| 04.XX | ✅ 배포완료 | ... | ... | ... |
| 04.XX | 🔜 배포예정 | ... | ... | ... |
```

### Step 3: 과제 이름별 진행 사항 통합 정리

개발팀 위클리 페이지의 **"수행 과제"** 컬럼과 기획팀 위클리 페이지의 **"금주완료"** 컬럼을 각각 파싱한다.

**핵심: 과제 이름별로 개발/기획 양쪽 내용을 통합 정리한다.**

1. 개발팀 "수행 과제"에서 과제 이름과 상태를 추출
2. 기획팀 "금주완료"에서 과제 이름과 상태를 추출
3. 동일하거나 유사한 과제 이름을 기준으로 그룹핑하여 정리

**과제별 진행 사항 섹션 순서 및 이름:**
1. 결제
2. 정산
3. 캐시/선불
4. 컴플라이언스
5. 인프라

**상태 표기는 스티커(이모지) 형태로 작성:**
- ✅ 완료
- 🔄 진행 중
- 🧪 QA 중
- 🔍 검토 중
- 📊 분석 중
- 🔜 배포예정 (배포 테이블용)

**과제별 테이블에는 JDP 티켓 링크 칸을 추가:**
- 핀테크 JDP 보드(XPJPD 프로젝트)에서 각 과제와 연관된 대표 티켓을 검색하여 링크를 추가한다
- JQL 예시: `project = XPJPD AND summary ~ "키워드"`
- 연관 티켓이 없으면 `—` 으로 표기

출력 포맷:
```markdown
## 과제별 진행 사항

### 결제

| 과제 | 상태 | 요약 | JDP 티켓 |
| --- | --- | --- | --- |
| OOO | ✅ 완료 | ... | [XPJPD-XXX](링크) |

### 정산
(동일 포맷)

### 캐시/선불
(동일 포맷)

### 컴플라이언스
(동일 포맷)

### 인프라
(동일 포맷)
```

통합 테이블 아래에는 과제명별 개발/기획 통합 내용도 포함한다:

```markdown
### [과제명]
- **기획**: (기획팀 금주완료 내용 요약)
- **개발**: (개발팀 수행과제 내용 요약)
- **상태**: ✅ 완료 / 🔄 진행 중 등 (스티커 형태)
- **담당**: 기획-OOO, 개발-OOO
```

그 아래에 작성자별 상세 내용도 포함한다:

- 작성자별 테이블의 다중 항목은 콤마(,) 기준으로 분리한다.
- 분리된 항목은 `* 항목` 형태로 쓰고, 같은 셀 안에서는 `<br>`로 줄바꿈한다.

```markdown
## 개발팀 작성자별 수행 과제

### 이름 (파트, 업무강도 N)

| 과제 | 상태 |
|------|------|
| ... | ... |

## 기획팀 작성자별 금주 완료

### 이름 (업무강도 N)

| 과제 | 상태 |
|------|------|
| ... | ... |
```

### Step 6: JDP 티켓에 위클리 요약 댓글 자동 추가

Step 3에서 JDP 티켓(XPJPD-XXX)이 매칭된 과제에 대해 해당 Jira 이슈에 댓글을 자동으로 추가한다.

#### 6-1. 대상 선정
- 과제별 테이블에서 `JDP 티켓` 칸에 XPJPD 키가 있는 과제만 대상
- `—` (매칭 없음)인 과제는 건너뛴다

#### 6-2. 댓글 내용 포맷
```markdown
**[위클리 요약] {대상날짜}** `상태스티커`

{과제 요약 내용}

- **기획**: {기획팀 내용} (해당 시)
- **개발**: {개발팀 내용} (해당 시)
- **담당**: {담당자}

> 출처: [핀테크트라이브 위클리 요약]({Confluence 요약 페이지 URL})
```

#### 6-3. 댓글 추가 실행
`addCommentToJiraIssue` 사용:
- `cloudId`: `kurly0521.atlassian.net`
- `issueIdOrKey`: XPJPD 티켓 키 (예: `XPJPD-159`)
- `commentBody`: 위 포맷의 댓글
- `contentFormat`: `markdown`

동일 티켓이 여러 과제에 매칭된 경우, 하나의 댓글로 합친다.
