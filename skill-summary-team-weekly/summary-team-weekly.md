핀테크트라이브 개발/기획팀 위클리를 요약하여 로컬 파일로 저장한다.

## 실행 절차

### Step 1: Confluence에서 대상 페이지 찾기

#### 1-1. 개발팀 위클리 페이지 찾기

개발팀 위클리 폴더 parentId: `5565612039`, site: `kurly0521.atlassian.net`

1. `getConfluencePageDescendants`로 하위 페이지 목록 조회 (depth: 1, limit: 10)
2. 제목 형식: `[개발] 26년 MM/DD - 26년 MM/DD`
3. 대상 날짜가 제목의 날짜 범위에 포함되는 페이지를 찾는다
4. `getConfluencePage` (contentFormat: markdown)으로 본문을 읽는다

#### 1-2. 기획팀 위클리 페이지 찾기

기획팀 위클리 폴더 parentId: `3963650997`, site: `kurly0521.atlassian.net`

1. `getConfluencePageDescendants`로 월별 하위 폴더 목록 조회 (depth: 1, limit: 5)
2. 대상 날짜의 월에 해당하는 폴더를 찾는다 (예: `26.04월 핀테크 트라이브 기획파트`)
3. 해당 월 폴더의 `getConfluencePageDescendants`로 주별 페이지 목록 조회 (depth: 1, limit: 10)
4. 제목 형식: `[기획]'26년 M/DD-M/DD`
5. 대상 날짜가 제목의 날짜 범위에 포함되는 페이지를 찾는다
6. `getConfluencePage` (contentFormat: markdown)으로 본문을 읽는다

### Step 2: 향후 2주내 배포 예정 정보 생성

개발팀 위클리 페이지의 테이블에서 **"향후 2주내, 예정된 주요 배포 내역"** 컬럼을 파싱한다.

- "배포 예정"으로 표기된 항목을 우선 정리
- "배포 완료"로 표기된 항목은 별도 섹션으로 분리
- 배포일 기준 오름차순 정렬

출력 포맷:
```markdown
## 향후 2주내 배포 예정

| 배포일 | 서비스 | 담당자 | 배포 기능 | QA |
|--------|--------|--------|-----------|-----|
```

### Step 3: 과제 이름별 진행 사항 통합 정리

개발팀 위클리 페이지의 **"수행 과제"** 컬럼과 기획팀 위클리 페이지의 **"금주완료"** 컬럼을 각각 파싱한다.

**핵심: 과제 이름별로 개발/기획 양쪽 내용을 통합 정리한다.**

1. 개발팀 "수행 과제"에서 과제 이름과 상태를 추출
2. 기획팀 "금주완료"에서 과제 이름과 상태를 추출
3. 동일하거나 유사한 과제 이름을 기준으로 그룹핑하여 정리

출력 포맷:
```markdown
## 과제별 진행 사항 (개발 + 기획 통합)

### [과제명]
- **기획**: (기획팀 금주완료 내용 요약)
- **개발**: (개발팀 수행과제 내용 요약)
- **상태**: 진행 중 / 완료 / 예정 등
- **담당**: 기획-OOO, 개발-OOO
```

그 아래에 작성자별 상세 내용도 포함한다:

- 작성자별 테이블의 다중 항목은 콤마(,) 기준으로 분리한다.
- 분리된 항목은 `* 항목` 형태로 쓰고, 같은 셀 안에서는 `<br>`로 줄바꿈한다.

```markdown
## 개발팀 작성자별 수행 과제

### 이름 (파트, 업무강도 N)

| 과제 | 상태 |
|------|------|
| ... | ... |

## 기획팀 작성자별 금주 완료

### 이름 (업무강도 N)

| 과제 | 상태 |
|------|------|
| ... | ... |
```
