**1-B. 외부 정보 폴더 (`infomation/`) 스캔 — 오늘 기준 -3일 이내 파일만**
```bash
find /Users/sy.im/Documents/내창고/주식/infomation -type f -mtime -3 -not -name '.*' | sort
```
- `_summary_*.md` 학습 노트가 있으면 **이를 먼저 읽고** 핵심 시그널 추출 (자금 흐름·테마 분석·신규 종목 후보)
- 학습 노트가 없거나 오래되었으면 원본 파일(이미지·텍스트)을 직접 Read로 OCR/본문 추출
- 추출한 시그널은 후속 Step의 4소스 종합분석에 추가 컨텍스트로 합침

**스크린샷 추출 규칙**:
- 빨간색(양수) 평가손익만 "수익 중"으로 분류. 파란색(음수)은 제외.

- 같은 종목의 현금/융자 포지션 또는 매수 시점이 다른 융자 포지션은 별도 행으로 보존 (집계 시 합산 X).
- 평가손익이 정확히 0인 포지션(±0.00%)은 수익에서 제외.

  - 출처는 실제 기사·공시 원문 URL (네이버뉴스·한경·이데일리·DART·KIND·로이터·블룸버그 등)

  - **중기(1~3개월) 방향성 시그널**: HBM·변압기·원전·밸류업 같은 큰 산업 사이클 + 매크로 리스크

  - **자금 빠지는 테마**: 한 줄 사유 + 보유 종목 매핑
  - 검색 키워드 예: "코스피 단기 전망 5월", "외국인 수급 한국 증시", "테마주 자금 흐름", "섹터 로테이션", "증시 전문가 전망"

- **생성/갱신 직전에 현재 시각 확보**: `date "+%Y-%m-%d %H:%M KST"` 또는 `date "+%Y-%m-%d %H:%M:%S %Z"` 실행 결과를 본문 최상단 메타 라인에 삽입 (아래 "노션 페이지 형식" 메타 라인 참조)

## 노션 페이지 형식 (마크다운)

**중복 금지·가독성 우선 원칙**:
- 같은 정보는 한 곳에서만 다룬다 (예: 종목 호재는 "호재/악재 표"에서만, 시장 환경은 "주요 시장 지표"+"어제 무슨 일"에서만).
- 액션·익절·손절·모멘텀은 모두 "내 종목 액션 플랜" 한 섹션에 통합한다.
- 모든 표는 정렬·헤더 중복 없이 한 번만 사용. 좁은 화면에서도 가독성 유지를 위해 컬럼은 최대 7개.
- 면책 조항은 **사용하지 않는다**. 출처는 페이지 **가장 마지막** 섹션에 위치.
- 강조는 **굵은 글씨**·이모지(🚨⚡🟢🟡⚠️) 활용해 시각적 구분. 색상 단어("빨간색/파란색")는 최소화.

**🚨 노션 표 작성 규칙 (절대 원칙)**:
노션은 마크다운 파이프 표(`| col | col |`)의 **정렬 구분자 행(`| --- | ---: |`)을 별개의 데이터 행으로 인식해 빈 행처럼 표시한다.** 따라서 모든 표는 아래 두 가지 중 하나로 작성:

1. **권장**: HTML `<table header-row="true">` 태그 사용
   ```html
   <table header-row="true">
   <tr>
   <td>헤더1</td>
   <td>헤더2</td>
   </tr>
   <tr>
   <td>값1</td>
   <td>값2</td>
   </tr>
   </table>
   ```
2. **마크다운 파이프 표를 사용할 경우**: **정렬 구분자 행을 절대 포함하지 말 것.** 헤더 행 다음에 바로 데이터 행이 오도록 작성:
   ```markdown
   | 헤더1 | 헤더2 |
   | 값1 | 값2 |
   ```
   (`| --- | ---: |` 같은 줄은 **절대 금지**)

페이지 작성 후에는 `notion-fetch`로 렌더링 결과를 검증해 빈 `---` 행이 들어가 있는지 확인할 것. 발견 시 `update_content`로 즉시 제거.

페이지 본문 순서는 아래와 같다 (총 12개 섹션 + 메타 라인).

### 0. `## 한눈에 보는 결론`

### 1. `## 주요 시장 지표` (현재 시점 기준)
**중요**: 전일 종가가 아니라 **매매전략 생성 시각 기준 가장 최신 값**을 사용. 장 중이면 실시간 또는 가장 최근 호가, 장 외면 직전 종가.

지표: 코스피, 코스닥, VKOSPI(코스피 변동성), VIX(미국 공포지수), 원달러(USD/KRW), 위안화(USD/CNY), WTI 유가, 브렌트유.

### 2. `## 오늘(M/D) 영향 줄 이슈 — 호재/악재 표`
오늘 장에 직접 영향을 줄 거시·정책·테마 이슈를 호재/악재로 분리. 종목별 개별 뉴스가 아니라 **시장 전체 또는 섹터 단위**로 묶어 일반 사용자가 한눈에 파악 가능하도록 작성.

#### 2-1. 🟢 호재 (오를 만한 이유)
컬럼: 이슈 / 설명(쉬운 말로) / 오늘 영향 받는 내 종목

#### 2-2. 🔴 악재 (조심할 이유)
컬럼: 이슈 / 설명 / 영향 받는 곳

> 환율·유가·정책·매크로·외국인 수급·"Sell in May" 같은 격언/통계 패턴까지 포함. 보유 종목 영향 명시 필수.

### 3. `## 어제 무슨 일이 있었나` (구어체)

#### 3-1. 시장 흐름 (3줄 요약)
- **미국 마감**: 지수 + 핵심 이슈 1줄
- **한국 마감**: 코스피·코스닥 + 외국인·기관 수급 1줄
- **매크로 이벤트**: FOMC·금리·환율·유가·지정학 중 핵심 1줄

#### 3-2. 🔥 어제 +10% 이상 급등한 내 종목 (필수)
보유 종목 중 직전 거래일 등락률 +10% 이상만 표로 정리. 해당 종목이 없으면 "어제 +10% 이상 급등한 보유 종목은 없어요" 한 줄 명기 (참고로 +9%대 종목은 비고 표시 가능).

#### 3-3. 💧 어제 -8% 이상 급락한 내 종목 (필수)
보유 종목 중 직전 거래일 등락률 -8% 이하만 표로 정리. 해당 종목이 없으면 "어제 -8% 이상 급락한 보유 종목은 없어요" 한 줄 명기.

> 급등락 사유는 WebSearch로 확인한 실제 뉴스·공시·반대매매·테마 모멘텀에 근거할 것. 추측 금지.

### 4. `## 증시 방향성 & 테마 자금 흐름` (예측·로테이션)
WebSearch로 수집한 인터넷 뉴스·증권사 리포트·전문가 칼럼·트레이더 발언을 종합해 **방향성과 자금 흐름**을 예측한다. 사용자가 단편적 권고 대신 큰 그림을 잡을 수 있도록 작성.

#### 4-4. 💧 자금 빠지는 테마

### 부록 A. `## 전문가팀 Q&A` (사용자 추가 질문 시에만, 본 형식 외)
사용자가 특정 종목·매크로 질문을 던지면 14인 패널 중 관련 전문가 3~5인이 짧게 발언 → 합의/분기 결론 → 사용자 액션. 일반 매매전략 보고서에는 포함하지 않음.

## 자주 나올 만한 추가 요청 (예측해서 미리 준비)
1. **"표로 만들어줘"** — 마크다운 표로 재정리
2. **"쉬운 말로 바꿔줘"** — 구어체로 다시 (영어 용어 한글 풀이 병기)
3. **"노션에 반영해줘"** — Notion MCP로 페이지 업데이트
4. **"출처 링크 넣어줘"** — Sources 섹션 추가/보강
5. **"특정 종목(예: 펄어비스) 분석"** — 14인 전문가팀 토론 형식으로 심층 분석
6. **"내일 매매전략"** — 시간대별 행동 가이드 + 시초가 액션 표
7. **"Q&A 추가"** — 전문가팀 토론 + 결론 + 사용자 액션
8. **"지표 추가"** — VKOSPI/VIX/환율/위안화/유가 표

## 결과 보고
전략 작성 후 노션 페이지 URL 한 줄을 사용자에게 알린다.
